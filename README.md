# MKR Signal Bot

Конструктор торговых «трейдеров»: правила входа по RSI / MACD / EMA на любых таймфреймах,
стоп/тейк по проценту цены или значению RSI, бэктест за неделю и месяц,
сигналы в Telegram-каналы с карточками-скринами. Работает полностью бесплатно:
Vercel (Hobby) + Neon Postgres + cron-job.org.

## Как это работает

- Веб-интерфейс — создание/редактирование/запуск трейдеров, статистика, бэктест.
- Раз в минуту внешний крон дёргает `/api/cron/tick`: проверяются открытые сигналы
  (стоп/тейк) и правила входа у запущенных трейдеров.
- При входе и закрытии сигнала бот шлёт в подключённые Telegram-каналы картинку
  со всеми настройками + текст с точными уровнями.
- Схема БД создаётся автоматически при первом запросе — миграции не нужны.
- Данные берутся с Binance Futures (`fapi.binance.com`) с фолбэком на спот-зеркало
  `data-api.binance.vision`. Регион функций Vercel — `fra1` (Binance блокирует US-IP).

## Деплой с нуля (~10 минут)

### 1. Telegram-бот

1. В Telegram открой [@BotFather](https://t.me/BotFather) → `/newbot` → задай имя и username.
2. Сохрани токен (`123456:ABC-...`).

### 2. Vercel

1. Запушь эту папку в GitHub (или `npx vercel` из папки проекта).
2. На [vercel.com](https://vercel.com) → **Add New → Project** → импортируй репозиторий.
   Фреймворк определится сам (Next.js), ничего менять не нужно.
3. В настройках проекта → **Environment Variables** добавь:
   - `TELEGRAM_BOT_TOKEN` — токен из BotFather
   - `CRON_SECRET` — любая случайная строка (например сгенерируй пароль)
   - `APP_PASSWORD` — (необязательно) пароль на веб-интерфейс
4. Deploy.

### 3. База данных (Neon, бесплатно)

1. В проекте Vercel → вкладка **Storage** → **Create Database** → выбери **Neon** (Postgres) → Create.
2. Vercel сам добавит переменную `DATABASE_URL`.
3. Сделай **Redeploy** проекта (Deployments → ⋯ → Redeploy), чтобы переменная подхватилась.

Таблицы создадутся сами при первом открытии сайта.

### 4. Вебхук Telegram

Открой в браузере (подставь свой домен и секрет):

```
https://ТВОЙ-ПРОЕКТ.vercel.app/api/telegram/setup?secret=ТВОЙ_CRON_SECRET
```

Должно вернуть `ok: true` и имя бота. После этого добавь бота в свой канал/группу
**администратором** — канал зарегистрируется автоматически (смотри страницу «Каналы»).

### 5. Крон (cron-job.org, бесплатно)

Vercel Hobby умеет запускать кроны только раз в день, поэтому берём внешний:

1. На [cron-job.org](https://cron-job.org) создай job:
   - URL: `https://ТВОЙ-ПРОЕКТ.vercel.app/api/cron/tick?secret=ТВОЙ_CRON_SECRET`
   - Расписание: каждую минуту
   - Timeout: 30–60 сек
2. Готово — трейдеры начнут проверяться каждую минуту.

## Использование

1. **+ Создать** → 4 шага: монета (топ-20 Binance или свой символ) / направление / плечо / имя →
   базовый ТФ и правила входа (можно сколько угодно, сигнал = все правила сработали одновременно) →
   стоп-лосс → тейк-профит.
2. Новый трейдер создаётся **на паузе**. Сначала прогони **бэктест** (неделя/месяц) на его странице.
3. **▶ Запустить** — трейдер начнёт сигналить в подключённые каналы.
4. Статистика (сигналов / по тейку / по стопу / профит с плечом) — на главной и на странице трейдера.

### Логика правил

- **RSI** — триггер при RSI ≤ или ≥ заданного значения на выбранном ТФ.
- **MACD** — N «разворотных свечей»: для LONG бары гистограммы ниже нуля растут N свечей подряд
  (красные столбики уменьшаются), для SHORT — зеркально.
- **EMA** — цена выше/ниже EMA или пересечение EMA вверх/вниз.
- Один трейдер держит не больше одного открытого сигнала и входит не чаще одного раза
  на базовую свечу.
- Если в одной минутной свече задеты и стоп и тейк — консервативно засчитывается **стоп**.
- Профит считается как движение цены × плечо (без комиссий и проскальзывания).

## Локальный запуск

```bash
cp .env.example .env.local   # заполни DATABASE_URL (можно взять из Neon) и остальное
npm install
npm run dev                  # http://localhost:3777
```

Тик руками: `http://localhost:3777/api/cron/tick?secret=...`
